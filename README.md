# Криптоаналізатор шифру Цезаря

В ході роботи над фінальним проектом першого модуля, була написана програма для шифрування та дешифрування файлів шифром Цезаря. 

Програма приймає на вхід 3 аргументи: command - команда,
filepath - абсолютний(повний) шлях до файлу, який кодується,
key - ціле число, ключ для зсуву по алфавіту. Є три доступні варіанти команди: три доступні варіанти: ENCRYPT, DECRYPT, BRUTE_FORCE.

ENCRYPT - кодування вмісту файлу. Кодуються усі літери англійського алфавіту та символи '.', ',', '«', '»', '"', '\'', ':', '!', '?', ' '. Під час кодування кожному з символів вхідного співставляється новий символ, який перебуває на певному постійному числі позицій праворуч від нього в алфавіті. Цей зсув визначається аргументом key. Після цього закодований текст записується у файл з ім'ям тим самим, що і вхідний файл, але з поміткою [ENCRYPTED].

DECRYPTED] - процес зворотній до кодування. Відбувається циклічний зсув символів по алфавіту з постійним значенням ключа key, але в зворотньому напрямку. Розкодований текст записується у файл з ім'ям тим самим, що і вхідний файл, але з поміткою [DECRYPTED].

BRUTE_FORCE - злам шифру. Алгоритм перебирає всі варіанти ключа для вхідного тексту. Якщо для якось ключа ми отримаємо розкодований текст, у якому тричі підряд після коми йде пробіл чи перенос рядка, то з високою впевненістю можна стверджувати, що шифр було зламано і цей ключ є правильним. Якщо програма, перебравши усі ключи, не знаходить правильного, то вона записує у файл розкодовані тексти для всіх значень ключа і пропонує користувачу самостійно знайти правильний. Вихідний текст записується у файл з ім'ям тим самим, що і вхідний файл, але з поміткою [BRUTE_FORCE].

Кінцева програма зібрана у форматі jar. Сам jar файл розміщений в releases на github.
